🎓 Student Dropout Prediction System

📖 Initial Problem Statement

Many students discontinue their studies before graduation. The university wants to reduce dropout rates and approaches an AI team to develop a prediction system.

Challenge: The statement is not yet a well-defined AI problem.


🔍 Step 1: Understand the Business Problem

Questions for Academic Department

  • Current dropout ratio?
  • Since when has it increased?
  • Which departments are affected?

Questions for Student Welfare Team

  • What interventions currently exist?
  • How much time is available before intervention?

Questions for Management

  • Why is this important?
  • What are the financial and social impacts?

📊 University Responses

  • Current dropout rate = 18%
  • Desired dropout rate = 10%
  • Most dropouts occur during the first year
  • Counsellors can support only 200 students/semester
  • Early identification improves retention

🎯 Step 2: Define the Problem Precisely

❌ Poor Problem Statement

Students are dropping out.

✅ Better Problem Statement

Identify first-year students who are at high risk of dropping out at least one semester before withdrawal so that counsellors can intervene.

Question Answer
Who? First-year students
What? Predict dropout risk
When? One semester in advance
Why? Enable early intervention
Outcome? Reduce dropout rate

🤔 Step 3: Is AI Really Needed?

If a simple rule solves the problem, AI may not be required.

Use AI When:

  • Multiple factors interact
  • Manual rules are insufficient

📥 Data Acquisition¶

The first step of this project is to acquire a dataset that represents various academic, demographic, and personal factors influencing student performance. Since a real-world dataset containing all the required attributes was not publicly available, a synthetic dataset was generated using NumPy and Pandas.

🌸 Synthetic Data Generation¶

A total of 10,000 student records were created using random number generation with a fixed seed (np.random.seed(42)) to ensure reproducibility of the dataset.

The generated dataset contains the following features:

📌 Feature 📖 Description 🎲 Generated Values
Age Student's age 17 – 25 years
Gender Gender of student Male / Female
Attendance Attendance percentage 40% – 100%
CGPA Academic performance 4.50 – 10.00
Backlogs Number of pending subjects 0 – 7
FamilyIncome Annual family income ₹1,50,000 – ₹1,20,00,000
StudyHours Daily study hours 0.5 – 8 hours
Scholarship Scholarship status 0 = No, 1 = Yes
InternetAccess Internet availability 0 = No, 1 = Yes
Hostel Hostel residence 0 = No, 1 = Yes
MentalStress Stress level 1 – 10
PartTimeJob Part-time employment 0 = No, 1 = Yes

✨ Why Synthetic Data?¶

  • 📊 Simulates realistic student information.
  • 🔄 Enables reproducible experiments using a fixed random seed.
  • 🧩 Provides sufficient diversity for machine learning model development.
  • 🚀 Eliminates privacy and confidentiality concerns associated with real student records.

🤖 Data Modelling (Training)¶

🌸 Model Training¶

The dataset is divided into training and testing sets. The training dataset is used to fit each classification algorithm, enabling the models to identify patterns and relationships within the data.

The following classification algorithms were trained:

🤖 Algorithm 📖 Purpose
Logistic Regression Linear classification model for binary prediction.
K-Nearest Neighbors (KNN) Classifies samples based on the nearest neighboring observations.
Decision Tree Learns decision rules by recursively splitting the dataset.
Random Forest Ensemble of multiple decision trees for improved accuracy and robustness.
Support Vector Machine (SVM) Finds the optimal decision boundary that maximizes class separation.
Gaussian Naive Bayes Probabilistic classifier based on Bayes' Theorem with Gaussian distribution assumptions.
XGBoost Classifier Gradient boosting algorithm that builds highly accurate ensemble models.

⚙️ Training Workflow¶

The following steps were performed for every classification model:

  • 📂 Load the machine learning algorithm.
  • 🎓 Train the model using the training dataset (X_train, y_train).
  • 🧠 Learn the underlying patterns from the input features.
  • 🔮 Generate predictions for the unseen testing dataset (X_test).
  • 📊 Store the predicted results for model evaluation.

✨ Why Train Multiple Models?¶

  • 📈 Compare the performance of different algorithms.
  • 🎯 Identify the model with the highest predictive capability.
  • ⚖️ Evaluate both simple and ensemble learning techniques.
  • 🚀 Select the most suitable classifier for the final deployment.

📊 Model Evaluation & Testing¶

🌸 Performance Evaluation¶

Each trained classifier predicts the class labels for the testing data. Whenever available, prediction probabilities are also generated to calculate probability-based evaluation metrics such as ROC-AUC.

The following evaluation metrics were computed for every model:

📈 Metric 📖 Description
Accuracy Measures the overall percentage of correctly classified instances.
Precision Indicates how many predicted positive instances are actually correct.
Recall Measures the model's ability to correctly identify positive instances.
F1-Score Harmonic mean of Precision and Recall, providing a balanced performance measure.
ROC-AUC Score Evaluates the model's ability to distinguish between different classes across various thresholds.

⚙️ Evaluation Workflow¶

For every trained classification model, the following steps were performed:

  • 📥 Load the trained model.
  • 🔮 Predict class labels using the testing dataset.
  • 📊 Generate prediction probabilities (where supported).
  • 📈 Calculate Accuracy, Precision, Recall, F1-Score, and ROC-AUC.
  • 📋 Store the evaluation results for comparison across all models.

✨ Why Use Multiple Evaluation Metrics?¶

  • 🎯 Accuracy alone may not fully reflect model performance.
  • ⚖️ Precision and Recall provide deeper insights into classification quality.
  • 📊 F1-Score balances false positives and false negatives.
  • 🚀 ROC-AUC measures the model's discrimination capability across different decision thresholds.
  • 🏆 Comparing multiple metrics helps identify the most reliable and robust classifier.

from sklearn.metrics import (
    accuracy_score,
    precision_score,
    recall_score,
    f1_score,
    roc_auc_score
)

results = []

for name, model in models.items():

    # Train
    model.fit(X_train, y_train)

    # Predict class labels
    y_pred = model.predict(X_test)

    # Predict probabilities
    if hasattr(model, "predict_proba"):
        y_prob = model.predict_proba(X_test)[:, 1]
    else:
        # SVM without probability=True
        y_prob = y_pred

    results.append({
        "Model": name,
        "Accuracy": accuracy_score(y_test, y_pred),
        "Precision": precision_score(y_test, y_pred),
        "Recall": recall_score(y_test, y_pred),
        "F1 Score": f1_score(y_test, y_pred),
        "ROC AUC": roc_auc_score(y_test, y_prob)
    })
